# Dishonored 1 (2012) - Modding & Localization Guide

**Engine:** Unreal Engine 3 (Hybrid / Modified)
**Platform:** PC (Steam)
**Localization Target:** Thai

---

## 1. ข้อมูลพื้นฐานและ Engine Constraints
เกม Dishonored 1 ใช้ Unreal Engine 3 (มีพื้นฐานผสม UE2) แต่อุปสรรคที่ใหญ่ที่สุดในการ Mod ภาษาคือไฟล์แพ็คเกจ `.upk` ของเกมนี้ถูก Compile แบบ **Big-Endian Byte Order** ซึ่งแปลกกว่าเกม UE3 ทั่วไปที่มักเป็น Little-Endian
* **ผลกระทบ:** เครื่องมือดัดแปลงของ UE3 ปกติ (เช่น UDK ดั้งเดิม) จะไม่สามารถอ่านไฟล์ `.upk` ของเกมนี้ได้โดยตรง ทำให้ต้องสร้างสคริปต์ Python (`binary_be.py`, `upkreader_be.py`) ขึ้นมาเพื่อจัดการ Byteswapping โดยเฉพาะ

---

## 2. โครงสร้างไฟล์ภาษา (Localization)
ข้อมูลตัวหนังสือ ซับไตเติล และสคริปต์ภายในเกมจะถูกฝังอยู่ในไฟล์ `CookedPCConsole` และ `DLC` 
- **Format:** ข้อความในเกมใช้ระบบ **UTF-16LE**
- **String Length:** ในเอนจิ้น UE3 ความยาวของ String ที่เก็บตัวอักษร 2 Bytes (อย่าง UTF-16LE หรือภาษาไทย) จะต้องถูกตั้งค่าเป็น **ค่าติดลบ (Negative Number)** ในระดับไบนารี (เช่น สตริงยาว 10 ตัวอักษร ต้องเก็บค่า Length = -11 เพื่อเผื่อ Null Terminator)
- **เครื่องมือที่ใช้:** สคริปต์ `subedit.py` ทำหน้าที่คำนวณและเขียนไบนารีทับลงใน `.upk`

### ปัญหาที่พบบ่อย: ตัวอักษรล่ม (Corrupted Encoding)
ในขั้นตอนการแปลหรือการแยกไฟล์ออกมาเป็น `.yaml` หากโปรแกรม Text Editor บันทึกไฟล์ผิด Encoding หรือตัวสคริปต์เก่าไม่ได้ตั้งค่า `utf-8` ให้ถูกต้อง จะทำให้ไฟล์ `.yaml` มีอักขระประหลาด หรือกลายเป็นเครื่องหมาย `?????` รัวๆ
* **วิธีแก้:** ต้องกลับไปนำข้อความต้นฉบับภาษาอังกฤษ (หรือไฟล์แบคอัปที่ถูกต้อง) มาแทนที่ใน `.yaml` ก่อนสั่งแพ็คเข้า `.upk` อีกครั้ง การรันคำสั่งแพ็คซ้ำเมื่อไฟล์ต้นทางพังไปแล้ว จะทำให้เกมแครชหรืออ่านไม่ออก

## 3. การจัดการฟอนต์และ UI (Scaleform)
Dishonored 1 ใช้ระบบ **Scaleform GFx** (Flash/SWF) ในการเรนเดอร์หน้าต่าง UI และฟอนต์
เมื่อทำการแก้ไขฟอนต์ โปรแกรม **JPEXS Free Flash Decompiler** จะสามารถเปิดไฟล์ `.upk` / `.gfx` เพื่อเข้าดูและแก้ไขไฟล์ `.swf` ด้านในได้ **แต่จะไม่สามารถกด Save ทับลงกล่อง `.upk` เดิมได้** เพราะ JPEXS ไม่รู้จักโครงสร้างกล่องของ UE3 (จะติด Error: *This SWF file source does not allow to save changes*)

### ทางออกในการแก้ไขฟอนต์ (Font Injection)
มี 2 วิธีหลักในการยัดฟอนต์ภาษาไทย:
1. **วิธีแยกไฟล์ (Extract & Inject):** 
   - ใช้สคริปต์ `unpack.py` แกะ `.upk` ให้เป็นไฟล์เดี่ยว 
   - แก้ไขไฟล์ `.SwfMovie` ด้วย JPEXS แล้ว "Save As" ออกมา 
   - ตั้งชื่อเติม `_patched` ต่อท้าย แล้วใช้คำสั่ง `python patch.py [ไฟล์ต้นฉบับ.upk]` เพื่อยัดไฟล์ที่แก้แล้วกลับเข้าไปใน UPK และอัปเดต Byte Offset อัตโนมัติ
2. **วิธีรันสคริปต์อัตโนมัติ (Automated Tool):** 
   - ใช้สคริปต์ `fontEdit.py` ที่อยู่ใน `dishonored-toolkit` ซึ่งออกแบบมาเพื่อรันแปลงฟอนต์ `.ttf` สร้างเป็น Texture2D (`.dds`) และฝังเข้าไปในไฟล์ `.Font` ของเกมได้ทันทีโดยไม่ต้องง้อ JPEXS เลย

## 4. โครงสร้างโฟลเดอร์ทำงาน (Workspace Standard)
เพื่อความเป็นระเบียบและลดปัญหาความซ้ำซ้อน โปรเจกต์ Dishonored 1 จึงถูกจัดเรียง Workspace ใหม่เป็นระบบตัวเลขดังนี้ (อัปเดตล่าสุด มิ.ย. 2026):

* `00_Backups` - ไฟล์สำรองฉุกเฉิน
* `01_Original_UPK` - ไฟล์ `.upk` ภาษาอังกฤษแท้ๆ จาก Steam
* `02_Translations_Workspace` - **แหล่งแก้ไขคำแปล** เก็บโครงสร้างโฟลเดอร์สำหรับ `.yaml` ทั้งหมดที่แยกตามด่าน
* `03_Tools` - เก็บเครื่องมือ Python เฉพาะทาง (`dishonored-toolkit`)
* `04_Scripts` - เก็บสคริปต์ Batch/Automation เช่น `pack_all.py` (รันแพ็คกิ้งแบบถอนรากถอนโคน)
* `05_Fonts_Source` - ฟอนต์ภาษาไทยตั้งต้น (`.ttf`)
* `06_Release_Build` - **ผลลัพธ์สุดท้าย** สคริปต์จะโยนไฟล์ที่แพ็คเสร็จแล้วเข้ามาที่นี่ พร้อมนำไปลงทับในเกม

*วิธีอัปเดตงานแปลที่ง่ายที่สุด:* เข้าไปแก้คำในโฟลเดอร์ `02` และเปิด Terminal รันคำสั่ง `python 04_Scripts/pack_all.py` แล้วรอรับไฟล์แจกจ่ายในโฟลเดอร์ `06` ทันที
